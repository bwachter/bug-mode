#+TITLE: GitHub REST API — bug-mode reference
#+AUTHOR: bug-mode developers
#+OPTIONS: toc:t

This document describes the GitHub REST API endpoints and conventions used
by the bug-mode GitHub backend (~bug-backend-github.el~).  It is not a full
API reference; see https://docs.github.com/en/rest for that.

* Authentication

All requests use a Personal Access Token (PAT) supplied via the ~:api-key~
instance property (or ~:api-key-file~ pointing to a file containing the
token).  The token is sent as a Bearer token:

#+BEGIN_EXAMPLE
Authorization: Bearer <token>
#+END_EXAMPLE

All requests also include:

#+BEGIN_EXAMPLE
Accept: application/vnd.github+json
X-GitHub-Api-Version: 2022-11-28
#+END_EXAMPLE

** Required token scopes

| Scope        | Why                                             |
|--------------+-------------------------------------------------|
| ~repo~       | Read issues on private repositories            |
| ~read:org~   | List organisations the token belongs to        |
| ~read:user~  | Read the authenticated user's login name       |

Public repositories can be read without any scope.

* Base URL

The default base URL is ~https://api.github.com~.  Override it with the
~:url~ instance property for GitHub Enterprise Server deployments.

* Rate limits

The REST API allows 5000 requests per hour for authenticated requests and
60 per hour for unauthenticated ones.  Search endpoints have a separate
limit of 30 requests per minute (10 per minute for unauthenticated).

Relevant response headers:

| Header                    | Meaning                                  |
|---------------------------+------------------------------------------|
| ~X-RateLimit-Limit~       | Requests allowed in the window           |
| ~X-RateLimit-Remaining~   | Requests left in the current window      |
| ~X-RateLimit-Reset~       | Unix timestamp when the window resets    |

* Instance configuration

#+BEGIN_EXAMPLE elisp
(:bug-github
 (:type      . github)
 (:url       . "https://api.github.com")   ; optional, this is the default
 (:api-key   . "ghp_...")                  ; PAT, or use :api-key-file
 (:project-id . "owner/repo"))             ; see Project scopes below
#+END_EXAMPLE

** Project scopes

The ~:project-id~ property controls the default search scope.

| Value           | Meaning                                                  |
|-----------------+----------------------------------------------------------|
| ~owner/repo~    | Single repository.  Direct issue fetch by number works. |
| ~org-login~     | All repositories in an organisation.                    |
| ~user:login~    | All personal repositories of a user account.            |

* Endpoints used by bug-mode

** Authenticated user

#+BEGIN_EXAMPLE
GET /user
#+END_EXAMPLE

Returns the authenticated user's profile.  Used by ~bug--list-github-projects~
to add a "Personal (login)" entry to the project list.

Selected response fields:

| Field    | Type   | Description              |
|----------+--------+--------------------------|
| ~login~  | string | Username                 |
| ~name~   | string | Display name (may be nil) |
| ~email~  | string | Public email (may be nil) |

** User organisations

#+BEGIN_EXAMPLE
GET /user/orgs?per_page=100
#+END_EXAMPLE

Lists organisations the token has access to.  Used by
~bug--list-github-projects~ to populate the organisation entries.

Selected response fields per object:

| Field    | Type   | Description                          |
|----------+--------+--------------------------------------|
| ~login~  | string | Organisation login (used as ID)      |
| ~name~   | string | Display name (may be empty or null)  |

** Single issue

#+BEGIN_EXAMPLE
GET /repos/{owner}/{repo}/issues/{issue_number}
#+END_EXAMPLE

Fetches one issue by number.  Used by ~bug--fetch-github-bug~ when the
bug UUID is a ~repos/…/issues/N~ resource path.

** List repository issues

#+BEGIN_EXAMPLE
GET /repos/{owner}/{repo}/issues?state=open&per_page=100
#+END_EXAMPLE

Lists open issues for a single repository.  Used by
~bug--list-github-project-bugs~ when ~:project-id~ is ~owner/repo~.

The response is a JSON array of issue objects.

** Search issues

#+BEGIN_EXAMPLE
GET /search/issues?q=<query>&per_page=100
#+END_EXAMPLE

Full-text and qualifier-based search.  Used for org-scoped and user-scoped
project listings and for general bug searches.

Response shape:

#+BEGIN_EXAMPLE json
{
  "total_count": 42,
  "incomplete_results": false,
  "items": [ <issue objects> ]
}
#+END_EXAMPLE

Common search qualifiers used by bug-mode:

| Qualifier         | Example                      | Meaning                        |
|-------------------+------------------------------+--------------------------------|
| ~repo:~           | ~repo:owner/repo~            | Limit to one repository        |
| ~org:~            | ~org:my-org~                 | Limit to an organisation       |
| ~user:~           | ~user:octocat~               | Limit to a personal account    |
| ~is:open~         |                              | Only open issues               |
| ~is:issue~        |                              | Exclude pull requests          |
| ~is:pr~           |                              | Only pull requests             |

* Issue object

The GitHub Issues API returns a superset of the fields listed here.  Most
URL-type and internal fields are hidden from display in bug-mode; they are
listed in the "Hidden" section for reference.

** Visible fields

| API field           | Display name    | Type | Notes                                    |
|---------------------+-----------------+------+------------------------------------------|
| ~number~            | Issue #         | int  | Normalised to string                     |
| ~title~             | Title           | str  |                                          |
| ~state~             | State           | str  | ~open~ or ~closed~                       |
| ~state_reason~      | State reason    | str  | ~completed~, ~not_planned~, ~reopened~   |
| ~user~              | Reporter        | str  | Normalised from ~user.login~             |
| ~assignee~          | Assignee        | str  | Normalised from ~assignee.login~         |
| ~assignees~         | Assignees       | str  | Comma-separated logins from array        |
| ~labels~            | Labels          | str  | Comma-separated label names from array   |
| ~milestone~         | Milestone       | str  | Normalised from ~milestone.title~        |
| ~comments~          | Comments        | int  |                                          |
| ~author_association~ | Role           | str  | ~OWNER~, ~COLLABORATOR~, ~NONE~, …       |
| ~html_url~          | URL             | str  | Browser URL for the issue                |
| ~created_at~        | Created         | date | ISO 8601 timestamp                       |
| ~updated_at~        | Updated         | date | ISO 8601 timestamp                       |
| ~closed_at~         | Closed          | date | ISO 8601 timestamp; nil when open        |
| ~locked~            | Locked          | bool | Hidden when false; shown as "Yes" if true |
| ~draft~             | Draft           | bool | Hidden when false; shown as "Yes" if true |
| ~Description~       | Description     | str  | Copied from ~body~ by normalise; rendered |
|                     |                 |      | in the description section at the bottom |

** Hidden fields

These fields are defined so the formatter knows their type, but they are
not shown in the issue buffer.

| API field                  | Type | Notes                                       |
|----------------------------+------+---------------------------------------------|
| ~body~                     | str  | Markdown; content copied to ~Description~   |
| ~id~                       | int  | Internal GitHub database ID                 |
| ~node_id~                  | str  | GraphQL node ID                             |
| ~url~                      | str  | API URL; used to derive ~_resource_path~    |
| ~repository_url~           | str  | API URL of the repository                   |
| ~labels_url~               | str  | URL template for labels                     |
| ~comments_url~             | str  |                                             |
| ~events_url~               | str  |                                             |
| ~timeline_url~             | str  |                                             |
| ~active_lock_reason~       | str  |                                             |
| ~reactions~                | obj  | Reaction counts object                      |
| ~pull_request~             | obj  | Present only when issue is linked to a PR   |
| ~performed_via_github_app~ | obj  |                                             |
| ~_resource_path~           | str  | Derived by bug-mode; used as bug UUID       |

** Bug UUID: ~_resource_path~

bug-mode uses ~_resource_path~ as the unique identifier for issues.  It is
derived from the ~url~ API field by stripping the base URL prefix:

#+BEGIN_EXAMPLE
url:            https://api.github.com/repos/owner/repo/issues/42
_resource_path: repos/owner/repo/issues/42
#+END_EXAMPLE

This self-contained path means issues fetched from a project list can be
opened without ~:project-id~ being configured in the instance.

* Pagination

bug-mode currently requests up to 100 results per page (~per_page=100~) and
does not follow pagination links.  For repositories or organisations with
more than 100 open issues only the first page is shown.

The Link response header provides ~rel="next"~ and ~rel="prev"~ URLs for
cursor-based pagination when additional pages exist.

* GitHub Enterprise Server

Point ~:url~ at your GHES API endpoint:

#+BEGIN_EXAMPLE elisp
(:url . "https://github.example.com/api/v3")
#+END_EXAMPLE

The same PAT-based authentication and endpoint structure apply; substitute
the GHES host and the ~/api/v3~ prefix for ~https://api.github.com~.
